Traveling Salesman Problem

Understanding the Traveling Salesman Problem (TSP)

The TSP asks: Given a set of cities and distances between them, what is the shortest possible route that visits each city exactly once and returns to the starting city?

Dynamic Programming with Bitmask

We use a bitmask to represent which cities have been visited. For n cities, each subset is represented by an n-bit integer.

dp[mask][i] = minimum cost to reach city i, having visited all cities in mask

How Path Cost is Calculated
  • Start at city 0 with mask = 1 (only city 0 visited)
  • Expand to each unvisited city, updating the mask
  • When all cities visited (mask = 2ⁿ - 1), add return cost to city 0
  • Total cost = sum of all edges in the tour
Example with 4 cities

Cities: A, B, C, D with distance matrix

Optimal Tour: A → B → D → C → A

Total Cost = d(A,B) + d(B,D) + d(D,C) + d(C,A)

Time Complexity

O(n² × 2ⁿ) - Exponential, but optimal for small n (≤ 20 cities)
